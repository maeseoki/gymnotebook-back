# Exercises Compatibility

| Area | Legacy behavior | New behavior | Preserved or changed | Reason | Frontend impact |
| --- | --- | --- | --- | --- | --- |
| Current user lookup | Routes resolved the user through the authenticated username. | Exercise routes use JWT `userId`. | Changed internally. | Avoids repeated username database lookups and handles username changes safely. | No response change. |
| List exercises | Returned exercises for the current user. Ordering was not explicit. | Returns only current user's exercises ordered by `name`, then `id`. | Preserved with deterministic ordering. | Stable API and tests. | Lists may appear sorted differently. |
| Ownership failure | Some route paths could return authorization errors after loading a record. | Missing and foreign-owned exercises both return `404 exercise_not_found`. | Intentionally changed. | Avoids leaking resource existence across users. | Clients should treat 404 as absent/unavailable. |
| Create body | Unknown fields were not explicitly rejected at the API contract layer. | Request bodies are strict; unknown fields such as `id` or `userId` are rejected. | Changed. | Prevents client-controlled persistence/ownership fields. | Clients must send only documented fields. |
| Text normalization | Legacy behavior depended on controller and validation behavior. | `name` and `description` are trimmed; blank descriptions become `null`; blank names are rejected. | Changed. | Consistent persistence and validation. | Whitespace-only names now fail validation. |
| Muscle groups | Legacy allowed values from enums; identical primary/secondary was not explicitly documented. | Same enum values; identical primary and secondary muscle groups are rejected. | Changed. | Duplicate primary/secondary muscle group is not useful domain data. | Clients must omit/clear secondary when it matches primary. |
| Image assignment | Exercise could reference an image without ownership enforcement. | New assignments require `image_data.user_id` to match the authenticated user. Missing, foreign-owned, and unresolved legacy images return `404 image_not_available`. | Changed. | Prevents attaching another user's images. | Existing unresolved references remain readable; new assignments require owned images. |
| Update response | Existing Fastify route returned `201` with no body. | Update returns `200` with the updated exercise response. | Changed. | Conventional update semantics and immediate response contract. | Clients should expect a response body. |
| Delete response | Existing Fastify route returned `204`. | Still returns `204` on success. If workout history references the exercise, returns `409 exercise_in_use`. | Preserved with explicit conflict. | Workout history is not cascade-deleted. | Clients should show a conflict instead of assuming delete succeeded. |
